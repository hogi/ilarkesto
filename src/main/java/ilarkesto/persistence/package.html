Framework for creating persistent objects (entities).

Entities & Data Access Objects

Every entity has a unique UUID (id). For every entity type there is a Data Access Object (DAO), named by the entity. While an entity represents the data, the DAO provides access to the entities. It contains a delete-, a save- and all available query-methods.

Transactions

When calling the delete- or the save-method for an entity on its DAO, the action is not performed instantly. Instead it is placed in the current transaction. The commit-method on the transaction needs to be called, to perform all save and delete actions to all entities involved in the transaction.

When changing properties of an entity, this entity is automatically placed in the current transaction. Calling the save-method on the DAO is only required when a new entity is created.

Data Store

Entities are saved as files. Each entity type gets its own folder. In this folder every entity is saved in its own file. The file is named by the entities id.

Backups

Every time an entity is modified or deleted, a backup copy of the old entity-file is created. A backup can by prevented for an entity type by implementing the BackupHostile-interface.
